通知优化

发表于 2017-09-15 | 分类于 Android

最近在对项目中的通知跳转逻辑进行优化，APP退出之后，从通知直接进入目标页面，之前会直接返回到桌面。为了优化用户体验，返回时应该跳转到APP主页面，实现方式有如下几种：

一、TaskStackBuilder

在AndroidManifest.xml配置Activity关系

<activity android:name=".MessageActivity"
      android:parentActivityName=".MainActivity"/>

代码

//启动通知Activity时，拉起主页面Activity
Intent msgIntent = new Intent();
msgIntent.setClass(this, MessageActivity.class);

//创建返回栈
TaskStackBuilder stackBuilder = TaskStackBuilder.create(this);
//添加Activity到返回栈
stackBuilder.addParentStack(MainActivity.class);
//添加Intent到栈顶
stackBuilder.addNextIntent(msgIntent);

PendingIntent pendingIntent = stackBuilder.getPendingIntent(0, PendingIntent.FLAG_UPDATE_CURRENT);

二、PendingIntent.getActivities()

代码：

Intent msgIntent = new Intent();
    Intent mainIntent = new Intent();
    msgIntent.setClass(this, MessageActivity.class);
    mainIntent.setClass(this, MainActivity.class);
    //注意此处的顺序
    Intent[] intents = new Intent[]{mainIntent, msgIntent};
    PendingIntent pendingIntent = PendingIntent.
            getActivities(this, 0, intents, PendingIntent.FLAG_UPDATE_CURRENT);

三、启动多个页面

1、过渡页中一次性启动多个页面

Intent[] intents = {mainIntent, msgIntent};
context.startActivities(intents);

2、在页面中依次按顺序启动页面，由传入参数控制

四、加一个flag，重构返回逻辑（实现简单）

代码：

Intent msgIntent =new Intent(context,MessageActivity.class);
//判断是否是点击notification后到的详细界面
startIntent.putExtra("notificationFlag",true);

重写BaseActivity的返回逻辑

notificationFlag =intent.getBooleanExtra("notificationFlag",false);
if(notificationFlag){
     Intent intent=new Intent(this,MainActivity.class);
     startActivity(intent);
     finish();
}

总结

在项目中，不会直接打开目标详情页，中间还需经过一个过度组件来准备数据，所以方案一、二排除。由于通知类型较多，需要处理较多的目标页面，方案三实现略微复杂。最后采用了方案四！
